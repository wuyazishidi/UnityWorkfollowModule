## 强制编译验证规则

修改任何 `.cs` 文件后，必须执行一次 Unity 编译验证。

## 验证命令

```powershell
powershell -ExecutionPolicy Bypass -Command "& '.\Packages\cn.etetet.yiuimcp\Config\compile-unity-flow.ps1' -Force 0 -NoWait 1"
```

需要强制编译时：

```powershell
powershell -ExecutionPolicy Bypass -Command "& '.\Packages\cn.etetet.yiuimcp\Config\compile-unity-flow.ps1' -Force 1 -NoWait 1"
```

## 当前脚本实际做的事

1. 读取 `UTO/.port`
2. 启动 UTO HTTP
3. 依次执行：
   - `StopPlayMode`
   - `TriggerCompile`
   - `GetCompileResult`
4. 输出编译结果
5. 关闭本次启动的 UTO 进程

## 处理规则

- 编译成功：继续后续工作
- 编译失败：先修复，再重新执行
- 不允许跳过编译验证直接声称完成

## 适用场景

- 新增 `.cs` 文件
- 修改现有 `.cs` 文件
- 删除 `.cs` 文件
- 修改类名、命名空间、工具参数、特性名或执行逻辑
